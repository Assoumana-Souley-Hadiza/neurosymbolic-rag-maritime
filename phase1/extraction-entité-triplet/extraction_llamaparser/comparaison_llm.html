🏆 MEILLEUR POUR LA QUALITÉ D'EXTRACTION

Claude 3.5 Sonnet

Avec 220 entités en 10 types distincts et 248 triplets RDF précis, Claude produit l'extraction la plus qualitative. Il distingue 57% de Prohibitions, 31% d'Obligations et 11% de Permissions — une nuance déontique absente des autres modèles. Les références légales exactes (§10(e) ICRW, Règle 15 MARPOL, Art. L.121-16) permettent une vérification directe. Seul point faible : 7 prédicats distincts contre 54 pour LlamaParser.

⚡Meilleur pour le volume + confiance

LlamaParser
Recommandé production

702 triplets, 54 prédicats mar:, confiance moyenne 0.94. Pour un triplestore en production avec couverture complète des 11 documents, LlamaParser est optimal. Les URI mar: normalisées sont directement ingestibles dans Apache Jena ou GraphDB.

🔬Meilleur pour la scalabilité locale

GLiNER + REBEL
Avec filtrage obligatoire

3 080 triplets 100% en local, coût nul, scalable à 100+ documents. Avec un seuil de confiance ≥0.7, on réduit le bruit de ~50%. Idéal pour un premier inventaire à grande échelle ou l'extraction d'espèces biologiques (298 entités de bonne qualité).

🏗️Architecture recommandée : Pipeline hybride

💡
La combinaison LlamaParser + Claude offre le meilleur rapport qualité/couverture pour un graphe de connaissances juridiques maritimes en production.
1. LlamaParser
→ Extraction d'entités verbatim par document, taxonomie complète (11 docs / 54 prédicats)
2. Claude 3.5
→ Qualification sémantique, triplets précis, types déontiques nuancés, citations d'articles (§§)
3. GLiNER (opt.)
→ Enrichissement espèces biologiques (298 entités) + inventaire rapide sur grands corpus
4. RDF Triplestore
→ Ingestion dans Apache Jena / GraphDB avec ontologie mar: — triplets dédupliqués
