Pourquoi ces criteres pour une ontologie ?
Une ontologie maritime necessite 5 proprietes fondamentales : fidelite (definition litterale, non paraphrasee), ancrage taxonomique (noms scientifiques pour les especes), tracabilite juridique (reference article/regle precise), completude semantique (exclusions pour borner les classes), et couverture (tous les documents, bilingue FR/EN).

Quel modele choisir pour notre ontologie ?
La question n'est pas quel modele est le meilleur en absolu, mais quel modele est le mieux adapte aux besoins specifiques d'une ontologie maritime. Notre critere cardinal : la fidelite verbatim.
LlamaParse
Meilleur — fidelite verbatim
Meilleur taux de fidelite (86%), definitions completes (100%), noms scientifiques inline. Ideal pour documents EN complexes. Limite : cloud, payant, aucun doc_id trace.
+ Verbatim le plus fiable (86%)
+ Definitions longues (209c moy.)
+ Noms sci. captures inline
- Cloud / payant / non reproductible
- Aucun doc_id trace
Gemma-4-9B-IT
Meilleur offline — profil ontologie
85% fidelite, noms scientifiques dans un champ dedie structure, precision reference 82%, exclusions capturees. Profil le plus equilibre pour un pipeline 100% offline.
+ Noms sci. champ dedie structure
+ Precision reference max (82%)
+ Exclusions capturees (9 entrees)
- Volume limite (96 def.)
- Licence HuggingFace a accepter
Mistral-8B
Meilleur — couverture corpus
Seul modele a couvrir les 9 documents. Rapide et stable sur T4. 66% de fidelite, bon travail de base. Insuffisant seul : aucune exclusion, tres peu de noms scientifiques.
+ Couverture 9/9 documents
+ Tres rapide sur T4
+ Apache 2.0, aucun token requis
- Zero exclusion, 3 noms sci. seulement
- Definitions courtes (132c moy.)
Qwen3-8B
Meilleur — volume brut
Volume inegale (669 definitions, 545 uniques). Multilingue natif. JSON strict reproductible. 62% fidelite — correct mais en dessous de LlamaParse et Gemma. Ideal en complement.
+ Volume maximal (669 defs)
+ Multilingue natif (29 langues)
+ JSON strict reproductible
- 158 defs paraphrasees trop courtes
- Zero nom scientifique extrait
Recommandation : pipeline hybride LlamaParse / Gemma + Qwen
LlamaParse pour la fidelite maximale quand le budget le permet. Gemma-4-9B-IT comme modele principal offline (precision, noms scientifiques structures, exclusions). Qwen3-8B en complement pour les ~388 termes supplementaires. Mistral-8B pour garantir qu'aucun document n'est manque. Deduplication post-traitement sur (terme.lower(), definition[:80]) pour fusionner sans doublon.
